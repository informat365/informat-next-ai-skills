# informat.jdbc Database Operations

## Overview

Use the `informat.jdbc` object to perform database-related operations
::: info

- To connect to MySQL, SQL Server, DB2, Oracle, and other databases, you need to download the corresponding JDBC driver.
- Directory: {Informat installation directory}/instance/informat-biz/libs
  :::

## createConnection

Create a database connection

```javascript
informat.jdbc.createConnection(connection)
```

::: tip

- An exception will be thrown if the connection cannot be created
- When `autoCommit` is set to `true`, transactions will be automatically committed when executing update, insert, or delete statements
- Errors during execution will automatically trigger a rollback
- Connections created will be automatically closed after the script execution ends
  :::

| Parameter | Type                                                       | Description |
|------------|----------------------------------------------------------|---------|
| connection | [ConnectionInfo](/guide/script/model.md#connnectioninfo) | Database connection information |

**Return Value**

Returns a database connection object [Connection](/guide/script/jdbc.md#connection), which can be used to perform CRUD operations on the database

**Example**

```javascript
const connection = informat.jdbc.createConnection({
    dburl: 'jdbc:oracle:thin:@host/XE',
    dbuser: 'user',
    dbpassword: 'pass',
    driverClassName: 'oracle.jdbc.driver.OracleDriver',
    autoCommit: false
})
connection.select('select * from table', (row) => {
    console.log(row.getString(1))
})
```

## Connection

### select

Execute a query. An exception will be thrown if the query fails

```javascript
connection.select(sql, handler, ...args)
```

| Parameter                                      | Type          | Description |
|----------------------------------------------|-------------|----------|
| sql                                          | `String`    | Query SQL |
| handler([ResultSet](/guide/script/jdbc.md#resultset)) | Function    | Query result callback function |
| args                                         | `...Object` | Query parameters |

**Example 1**

```javascript
// Query data from the user table where name equals tom
connection.select('select * from user where name=?', (rs) => {
    console.log(rs.getString('name'))
}, 'tom')
```

**Example 2 Using like**

```javascript
// Query data from the user table where name contains tom
connection.select('select * from user where name like concat(' % ',?,' % ')', (rs) => {
    console.log(rs.getString('name'))
}, 'tom')
```

**Example 3 Using in**

```javascript
// Query data from the user table where name is tom or jerry
connection.select('select name from user where name in(?,?)', (rs) => {
    console.log(rs.getString('name'))
}, 'tom', 'jerry')
```

### insert

Execute an insert. An exception will be thrown if the insert fails

```javascript
connection.insert(sql, returnAutoGeneratedKeys, ...args)
```

| Parameter                 | Type          | Description |
|-------------------------|-------------|-------------|
| sql                     | `String`    | Query SQL |
| returnAutoGeneratedKeys | `Boolean`   | Whether to return the auto-generated primary key ID |
| args                    | `...Object` | Insert parameters |

**Return Value**

Type is `Integer`

If returnAutoGeneratedKeys is set to true and the operation is an insert, returns the auto-increment ID of the last inserted data

**Example**

```javascript
connection.insert('insert into user(name,age) values (?, ?)', true, 'tom', 10)
```

### update

Execute an update or delete. An exception will be thrown if the update or delete fails

```javascript
connection.update(sql, ...args)
```

| Parameter | Type          | Description |
|------|-------------|--------|
| sql  | `String`    | Query SQL |
| args | `...Object` | Update parameters |

**Return Value**
Type is `Integer`

Returns the number of records successfully updated or deleted

**Example**
::: code-group

```javascript [Update]
connection.update('update user set age=?', 10)
```
```javascript [Delete]
connection.update('delete from user where age=?', 10)
```

:::

### commit

Commit a transaction. An exception will be thrown if the commit fails

```javascript
connection.commit()
```

### rollback

Rollback a transaction. An exception will be thrown if the rollback fails

```javascript
connection.rollback()
```

### callProcedure

Execute a stored procedure, applicable for scenarios with only IN parameters
```javascript
connection.callProcedure(sql, ...args)
```

| Parameter | Type          | Description |
|------|-------------|-----------------|
| sql  | `String`    | Stored procedure call statement |
| args | `...Object` | IN parameters (passed in order, supports null) |

::: tip
- `callProcedure` only supports IN parameters and does not return OUT parameter values
- Suitable for scenarios that only execute operations without needing return values
- If you need to get OUT/INOUT parameter return values, please use `callProcedureFull`
:::

**Example 1 - Stored procedure without parameters**
```javascript
var conn = informat.jdbc.systemConnection();
// Call a stored procedure without parameters
conn.callProcedure('CALL update_all_status()');
```

**Example 2 - Stored procedure with IN parameters**
```javascript
var conn = informat.jdbc.systemConnection();
// Call a stored procedure with parameters, passing them in order
conn.callProcedure('CALL insert_user(?, ?, ?)', 'tom', 25, 'developer');
```

**Example 3 - PostgreSQL type casting**
```javascript
var conn = informat.jdbc.systemConnection();
// PostgreSQL requires explicit type casting
conn.callProcedure('CALL update_user_status(?, CAST(? AS INTEGER))', 'u001', 1);
```

### callProcedureFull

Execute a stored procedure with support for IN, OUT, INOUT parameters and result set handling
```javascript
connection.callProcedureFull(sql, params, handler)
```

| Parameter | Type       | Description |
|---------|----------|----------------------------|
| sql     | `String` | Stored procedure call statement |
| params  | List<[ProcParam](/guide/script/model.md#procParam)>  | Parameter array (supports IN/OUT/INOUT) |
| handler([ResultSet](/guide/script/jdbc.md#resultset)) | `Function` | Result set callback function (**required**, handles returned result set data) |

::: warning Note
- **The handler parameter is required**, used to handle result sets returned by the stored procedure
- If the stored procedure does not return a result set, the handler function body can be null, but the parameter must be provided
- OUT/INOUT parameter return values are obtained through the Object returned by the method; result set data is processed through the handler
:::

**Parameter Object Structure**
```javascript
// IN parameter
{ 
    value: 'parameter value',     // Parameter value
    sqlType: 4,        // SQL type (required for PostgreSQL when the parameter value is numeric, otherwise explicit type casting is needed in SQL)
    isOut: false       // Marks as IN parameter
}

// OUT parameter (only supported by MySQL, Oracle, etc.; PostgreSQL does not support pure OUT)
{ 
    name: 'parameter name',      // Key name for the return value (optional)
    sqlType: 4,        // SQL type (required, see type table below)
    isOut: true,       // Marks as OUT parameter
    isOutOnly: true    // true = pure OUT
}

// INOUT parameter
{ 
    name: 'parameter name',      // Key name for the return value (optional)
    value: initialValue,       // Input value (can be null)
    sqlType: 4,        // SQL type (required)
    isOut: true,       // Marks as INOUT
    isOutOnly: false   // false = INOUT
}
```

**Return Value**

Type is `Object`, containing all OUT/INOUT parameter key-value pairs:
- If `name` is set, `name` is used as the key
- Otherwise `out1`, `out2`, etc. are used as keys

**handler Callback Function**
```javascript
function(resultSet) {
    // resultSet is a ResultSet object
    // You can use getString, getInt and other methods to get data
}
```

**Example 1 - MySQL: Basic INOUT Parameters**
```javascript
var conn = informat.jdbc.createConnection({
    dburl: 'jdbc:mysql://localhost:3306/mydb',
    dbuser: 'root',
    dbpassword: 'password',
    driverClassName: 'com.mysql.cj.jdbc.Driver'
});

var params = [
    { value: 1, isOut: false },  // IN parameter: status
    { 
        name: 'count',
        value: null,
        sqlType: 4,              // INTEGER
        isOut: true,
        isOutOnly: false         // INOUT parameter
    }
];

var result = conn.callProcedureFull(
    'CALL count_users_by_status(?, ?)',
    params,
    function(rs) {
        // If the stored procedure does not return a result set, the function body can be empty
    }
);

console.log('User count:', result.count);
```

**Example 2 - PostgreSQL: Multiple INOUT Parameters**
```javascript
var conn = informat.jdbc.systemConnection();

var params = [
    { value: 'u001', isOut: false },     // IN parameter: user ID
    { value: 50, isOut: false },         // IN parameter: points to add
    { 
        name: 'totalPoints',
        value: null,
        sqlType: 4,                      // INTEGER
        isOut: true,
        isOutOnly: false
    },
    { 
        name: 'message',
        value: null,
        sqlType: 12,                     // VARCHAR
        isOut: true,
        isOutOnly: false
    }
];

var result = conn.callProcedureFull(
    'CALL update_user_points(?, CAST(? AS INTEGER), ?, ?)', // The second IN parameter has no sqlType set, so explicit type casting is needed here
    params,
    null  // Can be set to null when there is no result set to process
);

console.log('Total points after update:', result.totalPoints);
console.log('Return message:', result.message);
```

**Example 4 - Processing Returned Result Sets**
```javascript
var conn = informat.jdbc.systemConnection();
var users = [];

var params = [
    { value: 1, sqlType: 4, isOut: false },  // IN parameter: status
    { 
        name: 'totalCount',
        value: null,
        sqlType: 4,
        isOut: true,
        isOutOnly: false
    }
];

// Use handler to process each row in the result set
var result = conn.callProcedureFull(
    'CALL get_users_with_count(?, ?)',
    params,
    function(rs) {
        // This function is called once for each row of data
        users.push({
            id: rs.getString('id'),
            username: rs.getString('username'),
            email: rs.getString('email'),
            age: rs.getInt('age'),
            points: rs.getInt('points')
        });
    }
);

console.log('Total users:', result.totalCount);
console.log('Found', users.length, 'users:');
users.forEach(function(user, index) {
    console.log('  ' + (index + 1) + '.', user.username, 
                '- Age:', user.age, '- Points:', user.points);
});
```

**Example 5 - Oracle: Using REF CURSOR**
```javascript
var conn = informat.jdbc.createConnection({

    dbuser: 'user',
    dbpassword: 'password',

var employees = [];

var params = [

```

**Example 8 - Processing Multiple Result Sets**
```javascript
var conn = informat.jdbc.createConnection({
    dburl: 'jdbc:mysql://localhost:3306/mydb',
    dbuser: 'root',
    dbpassword: 'password',
    driverClassName: 'com.mysql.cj.jdbc.Driver'
});
var topUsers = [];
var statsInfo = null;

var params = [
    { value: 1, sqlType: 4, isOut: false }
];

conn.callProcedureFull(
    'CALL get_user_summary(?)',
    params,
    function(rs) {
        // The handler automatically processes multiple result sets
        // You can determine the current result set by the number of columns
        var colCount = rs.getColumnCount();
        
        if (colCount === 3) {
            // First result set: id, username, points
            topUsers.push({
                id: rs.getString('id'),
                username: rs.getString('username'),
                points: rs.getInt('points')
            });
        } else if (colCount === 5) {
            // Second result set: statistics
            statsInfo = {
                totalCount: rs.getInt('total_count'),
                avgSalary: rs.getDouble('avg_salary'),
                totalPoints: rs.getInt('total_points'),
                maxAge: rs.getInt('max_age'),
                minAge: rs.getInt('min_age')
            };
        }
    }
);

console.log('Top users (top 5):', topUsers.length, 'people');
topUsers.forEach(function(user, idx) {
    console.log('  ' + (idx + 1) + '.', user.username, '- Points:', user.points);
});

if (statsInfo) {
    console.log('Statistics:');
    console.log('  Total:', statsInfo.totalCount);
    console.log('  Average salary:', statsInfo.avgSalary);
    console.log('  Total points:', statsInfo.totalPoints);
    console.log('  Age range:', statsInfo.minAge, '-', statsInfo.maxAge);
}
```

**Example 9 - SQL Server: With Return Value and OUT Parameters**
```javascript
var conn = informat.jdbc.createConnection({
    dburl: 'jdbc:sqlserver://localhost:1433;databaseName=mydb',
    dbuser: 'sa',
    dbpassword: 'password',
    driverClassName: 'com.microsoft.sqlserver.jdbc.SQLServerDriver'
});

var params = [
    { 
        name: 'returnValue',
        value: null,
        sqlType: 4,                      // INTEGER (RETURN value)
        isOut: true,
        isOutOnly: true
    },
    { value: 'zhangsan', isOut: false }, // IN parameter: username
    { value: 'password123', isOut: false }, // IN parameter: password
    { 
        name: 'userId',
        value: null,
        sqlType: 12,                     // VARCHAR
        isOut: true,
        isOutOnly: true
    },
    { 
        name: 'roleId',
        value: null,
        sqlType: 4,                      // INTEGER
        isOut: true,
        isOutOnly: true
    }
];

// SQL Server: Use {? = CALL ...} syntax to get RETURN value
var result = conn.callProcedureFull(
    '{? = CALL sp_user_login(?, ?, ?, ?)}',
    params,
    function(rs) {
        // If the stored procedure returns a result set, handle it here
    }
);

console.log('Return value:', result.returnValue);  // 0=success, -1=failure
console.log('User ID:', result.userId);
console.log('Role ID:', result.roleId);

if (result.returnValue === 0) {
    console.log('Login successful');
} else {
    console.log('Login failed');
}
```

::: tip Database Differences
**Parameter type support:**
- **PostgreSQL**: Only supports INOUT parameters, does not support pure OUT parameters, `isOutOnly` should be set to `false`. Supports returning result sets via cursor (sqlType: 1111)
- **MySQL**: Supports IN, OUT, and INOUT parameter types. Supports returning multiple result sets
- **Oracle**: Supports IN, OUT, INOUT, returns result sets via REF_CURSOR (sqlType: 2012)
- **SQL Server**: Supports IN, OUT, INOUT, and has RETURN value (must be the first parameter)

**SQL syntax differences:**
- PostgreSQL: `CALL procedure_name(?, ?)`
- MySQL: `CALL procedure_name(?, ?)`
- Oracle: `{CALL procedure_name(?, ?)}`
- SQL Server: `{CALL procedure_name(?, ?)}` or `{? = CALL procedure_name(?, ?)}`

**Type casting:**
- **PostgreSQL**: If `sqlType` is specified in params, there is no need to use `CAST(? AS type)` in SQL; the system will handle type conversion automatically
- **Other databases**: It is recommended to specify `sqlType` in params to avoid type conversion issues
:::

::: warning Important Notes
1. **handler parameter is required**: Even if the stored procedure does not return a result set, the handler function must be provided
2. **OUT/INOUT parameters must have sqlType set**: Refer to the "SQL Type Constants Table" above for type values
3. **Parameter naming**: Set the `name` property for OUT/INOUT parameters for easier access to return values
4. **NULL value handling**: The initial value of INOUT parameters can be null; handle null return values properly in JavaScript
5. **Multiple result set handling**: The handler function processes all result sets in order; you can use `rs.getColumnCount()` or column names to determine the current result set
6. **Cursor handling**: PostgreSQL and Oracle cursors are automatically expanded into row-by-row data in the handler
7. **Type specification advantage**: Specifying sqlType in params can simplify SQL statements and avoid manual type conversion
:::


## tableConnection

Get the database connection for application data tables

```javascript
informat.jdbc.tableConnection()
```

::: info

- Using the application data table database connection, you can query, update, and delete application data tables.
- An exception will be thrown if the connection cannot be created.
- When executing queries with the data table connection, you need to convert the data table identifiers to lowercase form
  :::

**Return Value**

Returns a database connection object [Connection](/guide/script/jdbc.md#connection), which can be used to perform CRUD operations on data tables in the database.

**Example**

For example, given a data table `tableDemo`

| Field Identifier | Name |
|--------|------|
| id     | ID   |
| name   | Name |
| maxAge | Max Age |

The following SQL should be used for querying

```sql
select id, name, max_age from table_demo;
```

```javascript
var ret = []
var conn = informat.jdbc.tableConnection();
conn.select(`select id,name,create_time from task limit 10`, (rs) => {
  ret.push({
    id: rs.getString('id'),
    name: rs.getString('name'),
    createTime: rs.getTimestamp('create_time'),
  })
});
console.log('ret', ret);
```

::: tip To query the `adjacency table` corresponding to a related list field, the query format is:

dataTableIdentifier$fieldIdentifier (camelCase converted to snake_case)
:::

**Example**

```javascript
var ret = []
var conn = informat.jdbc.tableConnection();
var sql = `select a.id from task$report a 
		inner join task b on a.id=b.id`;
conn.select(sql, (rs) => {
    ret.push(rs.getString('id'))
})
console.log('ret', ret);
```

## systemConnection

Get the system database connection

```javascript
informat.jdbc.systemConnection()
```

::: tip

- An exception will be thrown if the connection cannot be created. Using the system database connection, you can query, update, and delete all tables in the system.
- Using the system database connection, you can query, update, and delete all data tables in the system. Note that unless absolutely necessary, you should not use this feature. Informat's internal data tables may change with version updates, and directly accessing the underlying system data tables may cause application functionality issues after Informat version updates.
- Note that when executing SQL statements with `systemConnection`, the `appid` in the table name should be obtained using `informat.app.appId()`. The `appid` will change during application distribution.
  
:::

**Return Value**

Returns a database connection object [Connection](/guide/script/jdbc.md#connection)

**Example**

```javascript
var conn = informat.jdbc.systemConnection();
var ret = []
conn.select(`select id,name,create_time from v_croe0zft168y3_task limit 10`, (rs) => {
    ret.push({
        id: rs.getString('id'),
        name: rs.getString('name'),
        createTime: rs.getTimestamp('create_time'),
    })
})
console.log('ret', ret);
```

## safesql

Generate safe SQL by replacing placeholders `?` in the `sql` with safe complete SQL

```javascript
informat.jdbc.safesql(sql, params);
```

| Parameter | Type          | Description |
|--------|-------------|-------|
| sql    | `String`    | Database statement |
| params | `...Object` | Parameters |

**Return Value**
Type is `String`, safe SQL

**Example**

::: code-group

```javascript [Call]
const sql = `update tab set age=?,name=? where id=?`;
const params = [18, "李四", `'张' or 1=1`];
const result = informat.jdbc.safesql(sql, params);
```
```sql [Return Value]
update
  tab
set age  = 18,
    name = '李四'
where id = ''
  '张'
  ' or 1=1'
```
:::

## ResultSet

::: tip
When columnIndex is an index, the starting index is 1
:::

| Method                           | Return Type | Description |
|--------------------------------|---------|--------------------------|
